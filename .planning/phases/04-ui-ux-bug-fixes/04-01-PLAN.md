<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User keyboard/IME/clipboard → app text state | Untrusted local user input enters tool fields and is later parsed by crypto functions. |
| App text state → crypto execution | Field values become algorithm inputs, keys, or messages. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | `TextInputState` replacement APIs | mitigate | Keep all replacements bounded to current selection/range, clamp UTF-16 ranges to valid byte boundaries, and verify with `cargo test -p devtools`. |
| T-04-02 | Denial of Service | Multiline text inputs | accept | Local desktop tool; no network or persistence. Use normal GPUI rendering and right-content scrolling in Plan 03. |
| T-04-03 | Spoofing | Focus/error states | mitigate | Render blue focus border and red error border from explicit state so users can identify the active input. |
</threat_model>
